Reading Notes

Critical Web Design Ch. 0: Introduction

  • "critical design"challenges assumptions about existing designs by asking questions. Critical web design takes into account things like user experience and accessibility.
  • critical web design works to reveal qualities of human-computer interaction, also uncovering injustices or discrimination
  • critical web design can be found online and is intended to be functional and accessible. it involves interfaces/experiences like internet art, new approaches and ways of thinking.

Evolution of the Web

  • The history of the web we know today can be categorized into three phases - Web 1.0, Web 2.0, and Web 3.0
    • Web 1.0
      • ~1995
      • "read-only web" - websites were informational, static, and were linked together by hyperlinks
      • text-only e-mails could be written and sent
      • "content delivery network (CDN)" - users passively received the information shown on websites without interacting with it
    • Web 2.0
      • ~2005
      • "read-write web" / "participative social web" - growth in user interaction
      • blogging, social networking, music/video streaming, etc.
      • "Web as Platform" - software applications are built upon the web
      • Application Programming Interface/APIs - allow applications to communicate with each other
    • Web 3.0
      • ~2018
      • "executable web" / "read-write-execute web" - dynamic applications, interactive services, “machine-to-machine” interaction, sharing of data
      • "Semantic Web" -
        • AI and Machine Learning - machines process content in a human-like way, data is connected and understood contextually and conceptually
        • semantic metadata guides searches, affecting user experience
  • The future may hold a more universal, convenient, and decentralized web, with more secure data ownership and compensation for individuals.

The Wax and Wane of the Web

  • the web is constantly changing
  • designers and developers continue to learn to keep up with new standards
  • old standards like HTML are essential because they continue to be supported as evolution happens
  • it's important to optimize for performance and accessibility for the user, design with care, and explore beyond what's the most convenient
